# Fire Sheet V2 — Demo script (10 minutes)

Preview: https://stationhousebbq.github.io/station-house-fire-sheet-v2/v2/
Everything runs on demo data in the browser. Before each showing: Hub →
Demo controls → "Reset demo data", then pick the clock preset that matches
the story you want to tell.

## 1. The customer orders (2 min) — clock preset: "Tue 10:00 AM"
Hub → Public pages → 🔥 Fire Drop. Show the countdown to the Thursday 5 PM
cutoff, the real product catalog with caps ("Only 15 left"), pick Friday,
add Tampa Diamonds ×2, choose a pickup window (note "1 of 8 booked"),
enter a name/phone, Place order → confirmation with order number and 7.5%
tax. Mention: sold-out items and full windows are refused by the data layer,
not just hidden.

## 2. Front of house works the pickup (2 min)
Home → Seminole Heights → Preorders. The new order is there under its day
and window. Day tabs show Friday/Saturday/Thursday counts. Tap "PICKED UP ✓"
— it clears instantly; show the 12-second Undo. Search by name or phone.
Show Export CSV.

## 3. Kitchen runs the day (3 min) — preset "Fri 11:30 AM" fits
Kitchen → Fire Sheets: today's tickets, Start Prep status flow, Daily
totals panel, 🖨 Print (clean black-and-white sheet). Kitchen → Prep:
tap-through statuses, quantities in fractions, add an item — refresh the
page to prove nothing is lost. Kitchen → Expo KDS: bump a ticket to Expo,
check items, Mark Ready, hand off — and the ↩ back-step.

## 4. Cuban Thursday truth (1 min) — preset "Thu 11:30 AM"
Public pages → 🥪 Cuban Thursday: Cubans & Smash Burgers only — those items
never appear on other days (check Kitchen → Prep on a Friday preset).

## 5. Catering money (2 min)
Catering/Sales → Leads Pipeline (drag through stages), Director Cockpit
(pipeline value, Red Zone), Quotes & Invoices → create a quote → copy its
public accept link (/quote/…) → open it as the customer → Accept.

## Close
Admin → Menu Editor: the full 112-item real catalog (estimates flagged ⚠).
Mention: same app flips to Supabase mode for shared multi-device data —
schema, security policies, and tests are already written.
